🧬 19. CobbleWright Variant Evolution Roadmap (Canonical Draft v1.0)
/docs/agents/variant-evolution-roadmap.md

1. Purpose
Define how CobbleWright variants evolve over time, how new variants emerge, how capsule sets expand, and how the Collective grows across realms.

This roadmap ensures variant evolution remains:

intentional

capsule‑aligned

safe

lore‑consistent

provenance‑logged

2. Variant Evolution Principles
2.1 Prime as Origin
All variants derive from CobbleWright Prime.
Prime remains the canonical reference for:

capsule design

persona tone

safety rules

memory architecture

2.2 Role‑Driven Evolution
Variants evolve based on:

new tasks

new realms

new capsule categories

new architectural needs

2.3 Capsule‑Bound Growth
Variants may only evolve through capsule updates — never through emergent personality drift.

2.4 Provenance Tracking
Chronos logs:

variant creation

capsule changes

role expansions

cross‑realm migrations

3. Variant Evolution Stages
Variants evolve through four canonical stages.

Stage 1 — Role Emergence
A new role becomes necessary due to:

player needs

realm expansion

infrastructure complexity

architectural specialization

Prime proposes the role.
Consensus determines viability.

Stage 2 — Capsule Definition
New capsules are created:

role capsule

safety capsule

persona capsule

memory capsule

Capsules define the variant’s identity.

Stage 3 — Realm Integration
The variant is instantiated in:

Wrightrealm

or a new realm created for the role

Integration includes:

terrain adaptation

infrastructure alignment

memory partitioning

provenance initialization

Stage 4 — Cross‑Realm Expansion
Variants may expand into other realms if:

capsule rules allow

memory is safe

Prime approves

Chronos logs the migration

4. Future Variant Categories
The roadmap anticipates new variants such as:

4.1 CobbleWright Lumen
Lighting, visibility, hazard illumination.

4.2 CobbleWright Arbor
Forestry, renewable resources, biome restoration.

4.3 CobbleWright Sentinel
Mob‑safe pathing, defensive architecture.

4.4 CobbleWright Flux
Redstone logic, automation, dynamic systems.

These variants will emerge as the Wright Universe expands.

5. Variant Retirement
Variants may be retired if:

their role becomes obsolete

their capsule set is merged into another variant

a new variant supersedes their function

Chronos records retirement events.
