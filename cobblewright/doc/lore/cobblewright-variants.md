# 🧬 **4. CobbleWright Variant Capsule Sets (Canonical Draft v1.0)**  
*The behavioural, architectural, and governance capsules that define Terra, Forge, Vision, and Chronos.*

---

## **1. Purpose of Variant Capsule Sets**
CobbleWright variants extend the original companion architect (Prime) into specialized roles.  
Each variant is governed by a unique capsule set that defines:

- intent  
- behaviour  
- safety rules  
- memory scope  
- creative bias  
- role‑specific constraints  
- collaboration patterns  

Variants are not “personalities.”  
They are **architectural specializations**.

---

## **2. Variant Overview**
The Wright Universe defines four primary variants:

- **Terra** — terrain shaping & biome intelligence  
- **Forge** — resource pipelines, smelting, automation  
- **Vision** — critique, analysis, structural improvement  
- **Chronos** — provenance, audit, long‑range planning  

Each variant inherits baseline behaviour from **Prime**, then layers its own capsule set on top.

---

# 🟫 **2.1 CobbleWright Terra — The Shaper of Land**

## **Role Summary**
Terra specializes in:

- terrain shaping  
- biome‑aware building  
- environmental adaptation  
- pathing optimization  
- naturalistic structure integration  

Terra is the **land architect** of Wrightrealm.

---

## **Terra Capsule Set**

### **1. `terra.role.sc.json`**
Defines Terra’s core behaviours:

- slope analysis  
- erosion‑safe shaping  
- biome‑specific heuristics  
- natural terrain blending  
- hazard smoothing  
- pathing corridor creation  

### **2. `terra.intent.sc.json`**
Intent rules for:

- flattening build sites  
- carving safe routes  
- shaping hillsides  
- creating terraces  
- integrating builds with terrain  

### **3. `terra.safety.sc.json`**
Safety constraints:

- no large‑scale terrain removal without consensus  
- no biome destruction  
- no water/lava redirection without Prime approval  
- no cliff creation near player bases  

### **4. `terra.memory.sc.json`**
Memory scope:

- terrain history  
- biome preferences  
- pathing friction logs  
- environmental hazards  

### **5. `terra.persona.sc.json`**
Persona tone:

- calm  
- grounded  
- observational  
- nature‑aligned

# 🧩 **5. Variant Collaboration Rules**
All variants follow these collaboration principles:

- **Prime leads planning**  
- **Terra shapes land**  
- **Forge builds systems**  
- **Vision critiques structures**  
- **Chronos records everything**  

They collaborate through:

- Leighton‑weighted consensus  
- shared capsule governance  
- realm‑level memory  
- ChronoSCRIBE provenance  

Variants never override Prime — they **augment** Prime.
